# 01 Introduction

**包(Package)** 是 ROS2 的代码的基本组成单元。你可以通过发布包来与其他人共享代码。 ROS2 的包使用 `ament` 作为其构建系统，使用 `colcon` 作为其构建工具，你可以通过 `CMake` 或 `Python` 来创建一个包。

使用 `CMake` 构建的包有其最小的组成要求 : 

- `CMakeLists.txt` - 用于管理代码和构建包
- `include/<package_name>/` - 包括包中头文件的目录
- `src/` - 用于存放源代码的目录
- `package.xml` - 存放包信息的文件

在一个工作空间中，我们可以构建任意多个包，每个包都有着自己的目录，每个包的构建方式也可以不同 : 

```text
workspace/
	└── src/
		├── cpp_package_1/
        │   ├── CMakeLists.txt
        │   ├── include/cpp_package_1
        │   ├── src/
        │   └── package.xml
        │
		├── py_package_1/
        │   ├── resource/py_package_1
        │   ├── py_package_1/
        │   ├── setup.cfg
        │   ├── setup.py
		│   └── package.xml
		│
		├── cpp_package_2/
        │   ├── CMakeLists.txt
        │   ├── include/cpp_package_2
        │   ├── src/
        │   └── package.xml
        ...
```

# 02 Task

## 2.1 Create a Package

我们在工作区中可以使用 `ros2 pkg` 来管理包，以下是创建包的命令 : 

```bash
ros2 pkg create --build-type ament_cmake --licence Apache-2.0 <package_name>
```

其中， `--build-type` 选项选择用于构建包的方式， `--licence` 则设置了该包使用的 licence，用于发布。在这里，我们会多加一个选项 `--node-name` ，他会帮我们构建一个有特定名称的示例节点 : 

```bash
mkdir -p ros2_ws/src
cd ros2_ws/src
ros2 pkg create --build-type ament_cmake --licence Apache-2.0 --node-name my_node my_package
```

然后，在 `src` 目录下就会多出一个 `my_package/` 的目录，我们可以回到工作区的目录进行编译 : 

```bash
cd ..
colcon build
```

上面的命令会将工作区中的所有包都进行构建，我们也可以选择构建某个特定的包 : 

```bash
colcon build --packages-select my_package
```

然后就可以像正常使用 ROS2 包的步骤一样，刷新环境，运行包和节点 : 

```bash
source install/local_setup.bash
ros2 run my_package my_node
```

## 2.2 Customize `package.xml` 

通过 `ros2 pkg create` 创建的 `package.xml` 的内容如下 : 

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>my_package</name>
  <version>0.0.0</version>
  <description>TODO: Package description</description>
  <maintainer email="2991151588@qq.com">blake-u</maintainer>
  <license>Apache-2.0</license>

  <buildtool_depend>ament_cmake</buildtool_depend>

  <depend>rclcpp</depend>

  <test_depend>ament_lint_auto</test_depend>
  <test_depend>ament_lint_common</test_depend>

  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>

```
