## 2.2 Code

`cpp_pub.cpp` 的代码如下 : 

```C++
#include <iostream>
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>
#include <chrono>
#include <memory>

using namespace std::chrono_literals;

class MinimalPublisher : public rclcpp::Node
{
  public :
    MinimalPublisher()
    : Node("minimmal_publisher"), count_(0)
    {
      this->publisher_ = this->create_publisher<std_msgs::msg::String>("MyTopic", 10);
      auto timer_callback = 
        [this]() -> void
        {
          std_msgs::msg::String message = std_msgs::msg::String();
          message.data = "Hello World ! " + std::to_string(this->count_++);
          RCLCPP_INFO(this->get_logger(), "Publishing : '%s'", message.data.c_str());
          this->publisher_->publish(message);
        };
      
      this->timer_ = this->create_wall_timer(500ms, timer_callback);
    }

  private :
    rclcpp::TimerBase::SharedPtr timer_;
    rclcpp::Publisher<std_msgs::msg::String>::SharedPtr publisher_;
    size_t count_;
};

int main(int argc, char ** argv)
{
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<MinimalPublisher>());
  rclcpp::shutdown();

  printf("hello world cpp_pubsub package\n");
  return 0;
}
```

我们通过创建一个类继承自 `Node` 来管理 publisher，其中 : 

- `MinialPublisher` - 继承自 `rclcpp::Node` ，作为一个节点来管理 publisher
	- `publisher_` - 是存储在类中的 publisher，能够向指定话题发布数据
	- `timer_` - 是用于计时的计时器，能够在特定的时间之后执行某一指令，在这里，当时间达到 `500ms` 时，就会触发 `timer_callback` ，执行相应的代码
	- `timer_callback` - 则是一个 **lambda 表达式**，其中，我们创建了一个用于存储信息的 `message` ，并且通过 `RCLCPP_INFO` 向终端中输出发布的数据。最重要的是，通过调用 `this->publisher_->publish` 来向话题发布数据。
	
- `main` - 程序的主函数，用于运行节点
	- `rclcpp::init()` - 初始化程序
	- `rclcpp::spin()` - 通过接受 **某一节点的 SharedPtr** 来执行该节点
	- `rclcpp::shutdown()` - 在终端中按下 `Ctrl-C` 时结束节点的运行

## 3.2 Code

```C++
#include <iostream>
#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>

class MininalSubscriber : public rclcpp::Node
{
    public :
        MininalSubscriber()
        : Node("minimal_subscriber")
        {
            auto topic_callback = 
                [this](std_msgs::msg::String::UniquePtr msg) -> void
                {
                    RCLCPP_INFO(this->get_logger(), "I heard : '%s'", msg->data.c_str());
                };
            
            this->subscription_ = this->create_subscription<std_msgs::msg::String>("MyTopic", 10, topic_callback);
        }

    private :
        rclcpp::Subscription<std_msgs::msg::String>::SharedPtr subscription_;
};

int main(int argc, char* argv[])
{
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<MininalSubscriber>());
    rclcpp::shutdown();
    return 0;
}
```

subscriber 相对于 publisher 的代码比较简单，只需要订阅某个话题即可 : 

- `MinimalSubscriber` - 继承自 `Node` ，是我们自己写的节点
	- `subscription_` - 用于订阅话题的 subscriber，在接收到话题的数据后，会调用 `topic_callback` 来执行某些指令
	- `topic_callback` - **lambda 表达式** ，在这个表达式中我们向终端输出订阅得到的信息

> [!question] 
> 如果这个回调函数比较复杂，不打算使用 lambda 表达式，而是希望在这个类中实现一个回调的方法该如何做？
